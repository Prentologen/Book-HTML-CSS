Форми

HTML-форми є елементами управління, які застосовуються для збору інформації від відвідувачів веб-сайту.

Веб-формы состоят из набора текстовых полей, кнопок, списков и других элементов управления, которые активизируются щелчком мыши. Технически формы передают данные от пользователя удаленному серверу.

Для отримання і обробки даних форм використовуються мови веб-програмування, такі як PHP , Perl .

До появи HTML5 веб-форми представляли собою набір декількох елементів < input type = "text" > , < input type = "password" > , завершуються кнопкою < input type = "submit" > . Для стилізації форм в різних браузерах доводилося докладати чимало зусиль. Крім того, форми вимагали застосування JavaScript для перевірки введених даних, а також були позбавлені специфічних типів полів введення для вказівки повсякденному інформації типу дат, адрес електронної пошти та URL-адрес.

HTML5-форми вирішили більшість цих поширених проблем завдяки наявності нових атрибутів, надавши можливість змінювати зовнішній вигляд елементів форм за рахунок CSS3 .

Створення HTML5-форми

1. Елемент < form >

Основу будь-якої форми становить елемент < form > ... < / form > . Він не передбачає введення даних, так як є контейнером, утримуючи разом всі елементи управління форми - поля . Атрибути цього елемента містять інформацію, загальну для всіх полів форми, тому в одну форму потрібно включати поля, об'єднані логічно.

Таблиця 1. Атрибути тега < form >
Атрибут Значення / опис
accept-charset Значення атрибута являє собою розділений пробілами список кодувань символів , які будуть використовуватися для відправки форми, наприклад, < form accept-charset = "ISO-8859-1" > .
action Обов'язковий атрибут , який вказує url обробника форми на сервері, якому передаються дані. Являє собою файл (наприклад, action.php ), в якому описано, що потрібно робити з даними форми. Якщо значення атрибута не буде вказано, то після перезавантаження сторінки елементи форми приймуть значення за замовчуванням.
У разі, якщо вся робота буде виконуватися на стороні клієнта сценаріями JavaScript, то для атрибута action можна вказати значення # .
Також можна зробити так, щоб заповнена відвідувачем форма приходила вам на пошту. Для цього потрібно внести такий запис:
< form action = "mailto: адреса вашої електронної пошти" enctype = "text / plain" > < / form >
autocomplete Відповідає за запам'ятовування введених в текстове поле значень і автопідстановку їх при подальшому введенні:
on - означає, що поле не захищене, і його значення можна зберігати і витягувати,
off - відключає автозаповнення для полів форм.
enctype Використовується для вказівки MIME -типу даних, що відправляються разом з формою, наприклад, enctype = "multipart / form-data" . Вказується лише в разі method = "post" .
application / x-www-form-urlencoded - тип вмісту за замовчуванням, вказує на те, що передані дані представляють список URL-кодованих змінних форми. Символи пробілу (ASCII 32) будуть закодовані як + , а спеціальний символ, наприклад, такий як ! буде закодований шестнадцатиричной формі як % 21 .
multipart / form-data - використовується для відправки форм, що містять файли, які не-ASCII дані і бінарні дані, складається з декількох частин, кожна з яких представляє вміст окремого елемента форми.
text / plain - вказує на те, що передається звичайний (НЕ html) текст.
method Задає спосіб передачі даних форми.
Метод get передає дані на сервер через адресний рядок браузера. При формуванні запиту до сервера всі змінні і їх значення формують послідовність виду www.anysite.ru/form.php?var1=1&var2=2 . Імена та значення змінних приєднуються до адресою сервера після знака ? і розділяються між собою знаком & . Всі спеціальні символи і букви, відмінні від латинських, кодуються в форматі % nn , пробіл замінюється на + . Цей метод потрібно використовувати, якщо ви не передаєте великих обсягів інформації. Якщо разом з формою передбачається відправка будь-якого файлу, цей метод не підійде.
Метод post застосовується для пересилання даних великих обсягів, а також конфіденційної інформації і паролів. Дані, що відправляються за допомогою цього методу, не помітні в заголовку URL, так як вони містяться в тексті листа.
< form action = "action.php" enctype = "multipart / form-data" method = "post" > < / form >
name Задає ім'я форми , яке буде використовуватися для доступу до елементів форми через сценарії, наприклад, name = "opros" .
novalidate Вимикає перевірку в кнопці для відправки форми. Атрибут використовується без вказівки значення
target Вказує вікно, в яке буде направлено інформацію:
_blank - нове вікно
_self - той же фрейм
_parent - батьківський фрейм (якщо він існує, якщо немає - то в поточний)
_top - вікно верхнього рівня по відношенню до даного кадру. Якщо виклик відбувається не з дочірнього фрейму, то в той же фрейм.

2. Угруповання елементів форми

Елемент < fieldset > ... < / fieldset > призначений для угруповання елементів, пов'язаних один з одним, розділяючи таким чином форму на логічні фрагменти.

Кожній групі елементів можна привласнити назву за допомогою елемента < legend > , який йде відразу за тегом < fieldset > . Назва групи проявляється зліва у верхній межі < fieldset > . Наприклад, якщо в елементі < fieldset > зберігається контактна інформація: 

 < form >
  < fieldset >
    < legend > Контактна інформація < / legend >
    < p > < label for = "name" > Ім'я < em > * < / em > < / label > < input type = "text" id = "name" > < / p >
    < p > < label for = "email" > E-mail < / label > < input type = "email" id = "email" > < / p >
  < / fieldset >
< p > < input type = "submit" value = "Відправити" > < / p >
< / form >
Таблиця 2. Атрибути тега < fieldset >
Атрибут Значення / опис
disabled Якщо атрибут присутній, то група пов'язаних елементів форми, що знаходяться всередині контейнера < fieldset > , відключені для заповнення і редагування. Використовується для обмеження доступу до деяких полях форми, що містять раніше введені дані. Атрибут використовується без вказівки значення - < fieldset disabled > .
form Значення атрибута має дорівнювати атрибуту id елемента < form > в цьому ж документі. Вказує на одну або кілька форм, до яких належить дана група елементів. На даний момент атрибут не підтримує жодним браузером.
name Визначає ім'я , яке буде використовуватися для посилання на елементи в JavaScript, або для посилання на дані форми після заповнення і відправки форми. Є аналогом атрибута id .

3. Створення полів форми

Елемент < input > створює більшість полів форми. Атрибути елемента відрізняються в залежності від типу поля, для створення якого використовується цей елемент.

За допомогою css-стилів можна змінити розмір шрифту, тип шрифту, колір та інші властивості тексту, а також додати кордону, колір фону і фонове зображення. Ширина поля задається властивістю width .

Таблиця 3. Атрибути тега < input >
Атрибут Значення / опис
accept Визначає тип файлу, дозволених для відправки на сервер. Вказується лише для < input type = "file" > . Можливі значення:
file_extension - дозволяє завантаження файлів з зазначеним розширенням, наприклад, accept = ". gif" , accept = ". pdf" , accept = ". doc"
audio / * - дозволяє завантаження аудіофайлів
video / * - дозволяє завантаження відеофайлів
image / * - дозволяє завантаження зображень
media_type - вказує на медіа-тип завантаження.
alt Визначає альтернативний текст для зображень, вказується тільки для < input type = "image" > .
autocomplete Відповідає за запам'ятовування введених в текстове поле значень і автопідстановку їх при подальшому введенні:
on - означає, що поле не захищене, і його значення можна зберігати і витягувати,
off - відключає автозаповнення для полів форм.
autofocus Дозволяє зробити так, щоб в завантажується формі те чи інше поле введення вже мало фокус (було вибрано), будучи готовим до введення значення.
checked Атрибут перевіряє, чи встановлений прапорець за замовчуванням при завантаженні сторінки для полів типу type = "checkbox" і type = "radio" .
disabled Вимикає можливість редагування і копіювання вмісту поля.
form Значення атрибута має дорівнювати атрибуту id елемента < form > в цьому ж документі. Визначає одну або кілька форм, яким належить дане поле форми.
formaction Задає url файлу, який буде обробляти введені в поля дані при відправці форми. Здається тільки для полів типу type = "submit" і type = "image" . Атрибут переопределяет значення атрибута action самої форми.
formenctype Визначає, як будуть кодуватися дані полів форми при відправці на сервер. Перевизначає значення атрибута enctype форми. Здається тільки для полів типу type = "submit" і type = "image" . Варіанти:
application / & shy; x-www-form-urlencoded - значення за замовчуванням. Всі символи кодуються перед відправкою (прогалини замінюються на символ + , спеціальні символи перетворюються на значення ASCII HEX)
multipart / form-data - символи не кодують
text / plain - прогалини замінюються на символ + , а спеціальні символи не кодують.
formmethod Атрибут визначає метод, який браузер буде використовувати для відправки даних форми на сервер. Здається тільки для полів типу type = "submit" і type = "image" . Перевизначає значення атрибута method форми. Варіанти:
get - значення за замовчуванням. Дані з форми (пара ім'я / значення) додаються в url-адреса і відправляються на сервер: URL? Ім'я = значення & ім'я = значення
post - дані форми відправляються у вигляді http-запиту.
formnovalidate Визначає, що дані полів форми не повинні перевірятися при відправці форми. Перевизначає значення атрибута novalidate форми. Можна використовувати без вказівки значення атрибута.
formtarget Визначає, де виводити відповідь, отриманий після відправки форми. Здається тільки для полів типу type = "submit" і type = "image" . Перевизначає значення атрибута target форми.
_blank - завантажує відповідь в нове вікно / вкладку
_self - завантажує відповідь в той же вікно (значення за замовчуванням)
_parent - завантажує відповідь в батьківський фрейм
_top - завантажує відповідь на повний екран
framename - завантажує відповідь у фрейм із зазначеним ім'ям.
height Значення атрибута містить кількість пікселів без вказівки одиниці виміру. Встановлює висоту поля форми типу type = "image" , наприклад, < input type = "image" src = "img_submit.gif" height = "50" > . Рекомендується одночасно встановлювати як висоту, так і ширину поля.
list Є посиланням на елемент < datalist > , містить його id .Позволяет надати користувачеві кілька варіантів на вибір, коли він починає вводити значення у відповідному полі.
max Ця функція дозволяє обмежити допустимий введення числових даних максимальним значенням, значення атрибута може містити ціле або дробове число. Рекомендується використовувати цей атрибут разом з атрибутом min . Працює з наступними типами полів: number , range , date , datetime , datetime-local , month , time і week .
maxlength Атрибут задає максимальну кількість символів, що вводяться в поле. Значення за замовчуванням 524288 символів.
min Ця функція дозволяє обмежити допустимий введення числових даних мінімальним значенням.
multiple Дозволяє користувачеві ввести кілька значень атрибутів, розділяючи їх комою. Застосовується щодо файлів і адрес електронної пошти. Вказується без значення атрибута.
name Визначає ім'я, яке буде використовуватися для доступу до елементу < form > , наприклад, в таблицях стилів css. Є аналогом атрибута id .
pattern Дозволяє визначати за допомогою регулярного виразу синтаксис даних, введення яких повинен бути дозволений в певному полі. Наприклад, pattern = "[az] {3} - [0-9] {3}" - квадратні дужки встановлюють діапазон допустимих символів, в даному випадку - будь-які малі літери, число в фігурних дужках вказує, що потрібні три малі літери, після яких слід тире, далі - три цифри в діапазоні від 0 до 9.
placeholder Містить текст, який відображається в полі введення до заповнення (найчастіше це підказка).
readonly Чи не дозволяє користувачеві змінювати значення елементів форми, виділення і копіювання тексту при цьому є. Вказується без значення атрибута.
required Виводить повідомлення про те, що дане поле є обов'язковим для заповнення. Якщо користувач спробує відправити форму, не ввівши в це поле необхідне значення, то на екрані з'явиться попередження. Вказується без значення атрибута.
size Задає видиму ширину поля в символах. Значення за замовчуванням - 20. Працює з наступними типами полів: text , search , tel , url , email і password .
src Задає url зображення, що використовується як кнопки відправки даних форми. Вказується лише для поля < input type = "image" > .
step Використовується для елементів, які передбачають введення числових значень, вказує величину збільшення або зменшення значень в процесі регулювання діапазону (крок).
type button - створює кнопку.
checkbox - перетворює поле введення під прапорець, який можна встановити або очистити, наприклад,
У мене є автомобіль
color - генерує палітри кольорів в підтримуючих браузерах, даючи користувачам можливість вибирати значення квітів в шістнадцятковому форматі.
date - дозволяє вводити дату в форматі дд.мм.рррр.

file - дозволяє завантажувати файли з комп'ютера користувача.
Виберіть файл:
hidden - приховує елемент управління, який не відображається браузером і не дає користувачеві змінювати значення за замовчуванням.
image - створює кнопку, дозволяючи замість тексту на кнопці вставити зображення.
month - дозволяє користувачеві вводити рік і номер місяця за шаблоном рррр-мм.
number - призначене для введення цілочисельних значень. Його атрибути min , max і step задають верхній, нижній межі і крок між значеннями відповідно. Ці атрибути передбачаються у всіх елементів, що мають чисельні показники. Їх значення за замовчуванням залежать від типу елемента.

radio - створює перемикач - елемент управління у вигляді невеликого гуртка, який можна включити або виключити.
Вегетаріанець:
range - дозволить створити такий елемент інтерфейсу, як повзунок, min / max - дозволять встановити діапазон вибору
reset - створює кнопку, яка очищає поля форми від введених користувачем даних.
search - позначає поле пошуку, за замовчуванням поле введення має прямокутну форму.
Пошук:
submit - створює стандартну кнопку, що активізується клацанням миші. Кнопка збирає інформацію з форми і відправляє її для обробки.
text - створює текстові поля у формі, виводячи однорядкове текстове поле для введення тексту.
time - дозволяє вводити час в 24-годинному форматі за шаблоном чч: мм. В підтримують браузерах воно відображається як елемент управління у вигляді числового поля введення зі значенням, змінним за допомогою миші, і допускає введення тільки значень часу.
Вкажіть час:
url - поле призначене для вказівки URL-адрес.
Головна сторінка:
week - відповідний інструмент-покажчик дозволяє користувачеві вибрати один тиждень на рік, після чого забезпечить введення даних в форматі нн, ррр. Залежно від року число тижнів може бути 52 або 53.
Вкажіть тиждень:
value Визначає текст, що відображається на кнопці, в поле або пов'язаний текст. Чи не вказується для полів типу file.
width Значення атрибута містить кількість пікселів. Дозволяє задати ширину полів форми.

4. Текстові поля введення

Елемент < textarea > ... < / textarea > використовується замість елемента < input type = "text" > , коли потрібно створити великі текстові поля . Текст, що відображається як початкове значення, поміщається всередину тега. Розміри поля встановлюються за допомогою атрибутів cols - розміри по горизонталі, rows - розміри по вертикалі. Висоту поля можна задати властивістю height . Всі розміри вважаються виходячи з розміру одного символу моноширинного шрифту.

disabled Вимикає можливість редагування і копіювання вмісту поля.
form Значення атрибута повинно бути дорівнює значенню атрибута id елемента < form > в цьому ж документі. Визначає одну або кілька форм, яким належить дане текстове поле.

5. Список, що розкривається

Списки дають можливість розташувати велику кількість пунктів компактно. Розкриваються списки створюються за допомогою елемента < select > ... < / select > . Вони дозволяють вибрати одне або кілька значень із запропонованого безлічі. За замовчанням у переліку відображається його перший елемент.

Щоб додати до списку пунктів використовуються елементи < option > ... < / option > , які розташовуються усередині < select > .

Для систематизації списків застосовується елемент < optgroup > ... < / optgroup > , який створює заголовки в списках.

Для списків можливо змінити розмір шрифту, тип шрифту, колір та інші властивості тексту, а також додати кордону, колір фону і фонове зображення.

6. Написи до полів форми

Написи до елементів форми створюються за допомогою елемента < label > ... < / label > . Існує два способи угруповання написи і поля. Якщо поле знаходиться всередині елемента < label > , то атрибут for вказувати не потрібно. 

 <! - із зазначенням атрибута for - >
< label for = "comments" > Коли ви останній раз літали на літаку? < / label >
< textarea id = "comments" > < / textarea >

<! - без атрибута for - >
< p > < label > Кішка < input id = "cat" type = "checkbox" > < / label > < / p >

Атрибут Значення / опис

7. Кнопки

Елемент < button > ... < / button > створює клікабельні кнопки. На відміну від кнопок, створених < input > ( < input type = "submit" > < / input > , < input type = " image "> , < input type =" reset "> , < input type =" button "> ), всередину елемента < button > можна помістити контент - текст або зображення.

Для коректного відображення елемента < button > різними браузерами потрібно вказувати атрибут type , наприклад, < button type = "submit" > < / button > .

Кнопки дозволяють користувачам передавати дані в форму, очищати вміст форми або робити які-небудь інші дії. Можна створювати кордону, змінювати фон і вирівнювати текст на кнопці.

8. Прапорці та перемикачі в формах

Прапорці в формах задаються за допомогою конструкції < input type = "checkbox" > , а перемикач - за допомогою < input type = "radio" > .

Прапорців, на відміну від перемикачів, в одній формі може бути встановлено кілька. Якщо для прапорців вказано атрибут checked , то при завантаженні станиці на відповідних полях форми прапорці вже буде встановлено.

Елемент < label > застосовується при реалізації вибору за допомогою перемикачів і прапорців. Можна вибрати потрібний пункт, просто клацаючи кнопкою миші на тексті, пов'язаному з ним. Для цього потрібно помістити < input > всередину елемента < label > .
